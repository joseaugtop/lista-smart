<objective>
Implementar a PriceComparisonScreen completa (COMP-01..03, D-02/D-03/D-09) e criar os 4 arquivos de stub de teste Wave 0 exigidos pelo 03-VALIDATION.md. A tela consome `comparisonResultsProvider` (List<SupermarketTotal> ordenada ascendente, criada no Plan 01) e `fuelToggleProvider` (estado read-only aqui — o switch vive na ShoppingListScreen do Plan 03). Renderiza um card glassmórfico por supermercado, destaca o vencedor (index 0) com borda primary 2px + badge "Melhor opção", e mostra o breakdown produtos/combustível/total formatado em BRL, omitindo a linha de combustível quando o toggle está desativado.

Purpose: É o último passo do loop de compras (Home → Lista → Comparação) e a materialização da proposta de valor do app: "qual supermercado é mais barato para minha lista, incluindo deslocamento". Sem esta tela, o botão "Comparar Supermercados" do Plan 03 navega para uma rota vazia. Os stubs de teste Wave 0 fecham os gaps de validação pendentes do 03-VALIDATION.md para que a suite cubra os providers e a Home antes do gate de fase.
